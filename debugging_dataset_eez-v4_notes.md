# Debugging the EEZ Geometries Workflow

## EEZ workflow current method (without Argo):
1. manual: create geometry (writes to db.geometry table)
2. cli: cache command (writes zip to s3)
3. manual: make run id (we use this later, use fancy-long-uuid-thing but there cannot be any other runs in the db with this id)
3. cli: convert command (writes parquet/pmtiles to s3)
4. manual: create geometry run (leave provenance url and json empty!) (give this the run id that was made) (writes to db.geometry_run table)
5. cli: provenance. (writes json to s3 and to db.geometry_run table)

I need to debug the run id being written to the provenance. I also need to debug how to write to db.geometries_run.provenance_x fields. There is a circular dependency. Step 4 makes this db record, but step 5 makes the provenance. However step 5 cannot run until step 4 is completed because of DB schema fkey restrictions.

Then run the make command. It will trigger the breakpoints and you can use these commands to debug.

```
n (next): Step to the next line in the current function.
c (continue): Continue execution until the next breakpoint.
s (step): Step into a function call.
l (list): Show source code around the current line.
p <expression>: Print the value of an expression, e.g. p variable_name.
q (quit): Exit the debugger and terminate the program.
```
